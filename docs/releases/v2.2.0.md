# Codex Quota 2.2.0

Codex Quota now treats the weekly allowance as the current primary limit while remaining ready for a future short rolling window. The app follows the quota windows actually returned by Codex instead of assuming that a five-hour limit is always present.

Codex Quota 现在以每周额度作为当前主要限制，同时保留未来恢复短周期额度的兼容能力。应用会根据 Codex 实际返回的额度窗口展示数据，不再假定 5 小时限制始终存在。

## Install / 安装

### DMG

Download `Codex-Quota-macOS-universal.dmg`, open it, and drag **Codex Quota** into **Applications**.

下载 `Codex-Quota-macOS-universal.dmg`，打开后将 **Codex Quota** 拖入 **Applications（应用程序）**。

### One-command installer / 一行安装

```bash
curl -fsSL https://raw.githubusercontent.com/Zamisku/Codex-Quota/main/scripts/install-release.sh | /bin/zsh
```

The installer verifies the published SHA-256 checksum, code signature, application and widget Bundle IDs, backs up an existing installation, registers the widget, and launches the app.

安装脚本会核验公开的 SHA-256、代码签名、应用及小组件 Bundle ID，备份旧版本、注册小组件并启动应用。

## What changed / 更新内容

- Weekly-only responses now work in the host app, menu bar, Small widget, and Medium widget.
- Window duration takes precedence over legacy `primary` and `secondary` field names, preventing a seven-day window from being mislabeled as five hours.
- If Codex restores an 18,000-second short window, the existing dual-window presentation returns automatically.
- Weekly-only, short-only, dual-window, ambiguous legacy, unknown-field, stale-snapshot, and old-snapshot cases are covered by regression tests.
- The bilingual project introduction and promotional artwork now describe adaptive quota windows.

- 宿主应用、菜单栏以及 Small、Medium 小组件现已完整支持仅返回每周额度的响应。
- 窗口时长优先于旧版 `primary`、`secondary` 字段名，避免把七天窗口错误标记为 5 小时。
- 如果 Codex 重新返回 18,000 秒短周期窗口，双窗口界面会自动恢复。
- 回归测试覆盖 weekly-only、short-only、双窗口、旧版歧义字段、未知字段、旧数据回退和旧快照兼容。
- 中英文项目介绍与宣传素材已同步为额度窗口自适应口径。

## First-launch notice / 首次启动说明

This build is signed with Apple Development but is not notarized. When installing through Finder, Control-click the app, choose **Open**, and confirm once. The command-line installer downloads and verifies the same release without requiring Xcode.

当前构建使用 Apple Development 签名，但尚未完成 Apple 公证。通过 Finder 安装时，请按住 Control 点击应用，选择“打开”并确认一次。一行安装器下载并校验同一发行包，全程不需要 Xcode。

See `SHA256SUMS` in this release before installing manually.
